Development

Release notes

  • requires MooTools 1.45
  • (alpha) IE8 and below are not fully supported. Some features may work for you, but some are broken, including non-bubbling input events like focus/blur and submit.
  • Currently, the framework cannot used with advanced/unsafe compression methods: function renaming will break it. Advanced compression will be implemented later.

Compatibility

LiquidLava and Firestorm frameworks can not coexist with code that modifies the native object's ptototype.

Bugs

See the Tasks page and GitHub tracker.

If you found a bug or a mistake somewhere, please, submit an issue.

Versioning

Git master branch has tags 'version-x', which you may use to switch to specific framework version. See also the GitHub releases page.

In case of any breaking changes, there will be no backward compatibility. Instead, you will be given detailed upgrade instructions. See the Changelog.

Building

First you will need to run

npm install

in the project root to install dependencies, then

grunt

to build the current master.
